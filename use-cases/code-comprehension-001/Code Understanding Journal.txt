###Exercise Part 1: Understanding a Specific Feature

1. The main components involved in task creation/updates
cli.py, the file that direclty handles arguements passed in the terminal
task_manager.py, validates data and has functions that are responsible for creating and updating functionality. 
models.py, has a Task class that defines the structure of the task
storage.py, contains functions that interact with the json file where the tasks are stored

2. The execution flow when a task is created/updated

A command is typed using the interface handled by cli.py
The arguments are parsed to  a function in task_manager.py
When a task is created: 
task_manager.py calls the function create_task().
The Task class from models.py creates a task with a new UUID.
storage.py uses TaskEncoder for the information to be in a dictionary
Storage.py stores the task in a JSON file by calling json.dump().
tasks.json is updated.
When a task is updated: 
task_manager.py calls the function update_task().
storage.py loads the JSON file where all tasks are stored and finds a matching UUID.
storage.py uses TaskEncoder for the information to be in a dictionary
Storage.py stored the task in a JSON file by calling json.dump().
tasks.json is updated.


3. How data is stored and retrieved

Data is stored in a JSON file called tasts.json.
Storage.py stores the task in a JSON file by calling json.dump().
storage.py loads the JSON file where all tasks are stored by calling json.load().
